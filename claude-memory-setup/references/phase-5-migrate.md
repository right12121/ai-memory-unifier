# Phase 5 — Migrate sources into archive

This is the only destructive-ish phase, and even here we only `mv` — never `rm`. The rule is: **if something breaks, we can always put it back**.

## Archive directory layout

```
~/.claude/archive-<YYYY-MM-DD>/
├── projects/
│   └── <encoded-project-path>/
│       └── memory/
│           └── <source file>.md
├── openclaw-extract/        # optional, if user has OpenClaw-type third-party agent
├── misc/                    # scattered CLAUDE.md files, pre-reorg snapshots
├── removed-skills/          # only if this run removed any skills
├── rollback.sh              # generated script to undo the migration
└── <original-CLAUDE.md path>.pre-reorg    # if CLAUDE.md existed before
```

Keep the archive path structure similar to the source path so manual recovery is intuitive.

## Manifest format

`~/.claude/reorg-log/manifest-<YYYY-MM-DD>.json`:

```json
{
  "reorg_date": "2026-04-21",
  "reorg_version": "1.0",
  "source_tool_versions": {
    "claude_code": "<from claude --version>",
    "codex": "<if present>"
  },
  "files": [
    {
      "source": "/Users/<you>/.claude/projects/-Users-<you>/memory/user_role.md",
      "archive": "/Users/<you>/.claude/archive-2026-04-21/projects/-Users-<you>/memory/user_role.md",
      "sha256": "4e90512a...",
      "size": 1234,
      "mtime": "2026-04-15T09:12:34+08:00",
      "category": "identity",
      "decision": "migrated-to-claude-md",
      "decision_target": "~/.claude/CLAUDE.md#Identity",
      "migrated_at": "2026-04-21T10:23:11+08:00"
    }
  ],
  "summary": {
    "total_files": 27,
    "by_decision": {
      "migrated-to-claude-md": 6,
      "migrated-to-skill-feishu": 4,
      "new-skill-pencil-design": 4,
      "kept": 2,
      "archived-only": 11
    },
    "total_bytes": 54321
  }
}
```

Every entry has `sha256` so rollback is verifiable.

## Migration order (safety first)

1. **Verify manifest is fully written to disk** (can be re-read as valid JSON)
2. **Create archive parent directories** with `mkdir -p`
3. **Compute SHA256 once** before any move, and verify it matches the manifest entry
4. **Move in this order**:
   a. First: AutoMemory directories (`~/.claude/projects/*/memory/` — entire dirs at once)
   b. Then: scattered CLAUDE.md files (e.g., `~/mine/CLAUDE.md`)
   c. Then: any removed-skills directories
5. **Verify post-move**: source path no longer exists, archive path does, size matches manifest
6. **Update manifest** with `migrated_at` timestamp per file

## Use `mv`, not `cp + rm`

`mv` is atomic (usually). `cp + rm` creates a window where both copies exist, and a failure between them leaves the system in a weird state.

```bash
# Correct
mv "$src" "$archive_path"

# Wrong
cp "$src" "$archive_path" && rm "$src"
```

## Handling AutoMemory's tendency to regenerate

AutoMemory (if still enabled in settings) may re-create the moved files. That's fine — Loop 2 will catch them and suggest classification on the next daily run.

Do NOT disable AutoMemory during migration. Only disable if the user explicitly asks.

## Moving `~/mine/CLAUDE.md` (or similar scattered CLAUDE.md)

If a project-root CLAUDE.md was merged into the global CLAUDE.md:

```bash
mv ~/mine/CLAUDE.md ~/.claude/archive-2026-04-21/misc/mine_CLAUDE.md
```

Why move and not leave? If both `~/mine/CLAUDE.md` and `~/.claude/CLAUDE.md` exist, Claude Code will load both (walk-up merge), **duplicating content in every session from that directory**. Remove the duplicate.

If a project-root CLAUDE.md is intentionally kept (e.g., `~/axon-web/CLAUDE.md` for technical project docs), record it in the global CLAUDE.md's "Project-Level CLAUDE.md 保留清单" section, and don't move it.

## Generate `rollback.sh`

This script lives in the archive dir and knows how to put everything back:

```bash
#!/bin/bash
# Generated by claude-memory-setup on <date>
# Running this script restores the pre-reorg state.
# Execute from the archive dir: bash ~/.claude/archive-<date>/rollback.sh

set -euo pipefail

ARCHIVE=~/.claude/archive-<YYYY-MM-DD>

# 1. Restore AutoMemory directories
for proj_dir in "$ARCHIVE"/projects/*/; do
  proj_name=$(basename "$proj_dir")
  mkdir -p ~/.claude/projects/"$proj_name"
  mv "$proj_dir/memory" ~/.claude/projects/"$proj_name"/memory
done

# 2. Restore scattered CLAUDE.md files
[ -f "$ARCHIVE/misc/mine_CLAUDE.md" ] && mv "$ARCHIVE/misc/mine_CLAUDE.md" ~/mine/CLAUDE.md
# Add more as needed per manifest

# 3. Restore any removed skills
[ -d "$ARCHIVE/removed-skills" ] && for sk in "$ARCHIVE/removed-skills"/*/; do
  mv "$sk" ~/.claude/skills/$(basename "$sk")
done

# 4. Restore pre-reorg CLAUDE.md if present
if [ -f "$ARCHIVE/CLAUDE.md.pre-reorg" ]; then
  mv ~/.claude/CLAUDE.md ~/.claude/CLAUDE.md.post-reorg-backup
  mv "$ARCHIVE/CLAUDE.md.pre-reorg" ~/.claude/CLAUDE.md
fi

echo "Rollback complete. Pre-reorg state restored."
echo "Your post-reorg CLAUDE.md was saved as ~/.claude/CLAUDE.md.post-reorg-backup (if it existed)."
echo "Review the manifest at $ARCHIVE/../reorg-log/manifest-<date>.json for details."
```

Generate this dynamically based on what was actually moved. Save it `chmod +x`.

## Post-migration verification

Before declaring Phase 5 done:

```bash
# 1. All source paths from manifest should NOT exist anymore
for src in $(jq -r '.files[].source' manifest-<date>.json); do
  [ ! -e "$src" ] || echo "STILL EXISTS: $src"
done

# 2. All archive paths SHOULD exist
for arc in $(jq -r '.files[].archive' manifest-<date>.json); do
  [ -e "$arc" ] || echo "MISSING FROM ARCHIVE: $arc"
done

# 3. SHA256s still match
# (tedious but important for the first few to build trust)
for entry in $(jq -c '.files[:5][]' manifest-<date>.json); do
  expected=$(echo "$entry" | jq -r '.sha256')
  archive=$(echo "$entry" | jq -r '.archive')
  actual=$(shasum -a 256 "$archive" | awk '{print $1}')
  [ "$expected" = "$actual" ] && echo "OK: $archive" || echo "MISMATCH: $archive"
done
```

Emit a summary in chat:
> Phase 5 complete. Migrated N files totaling X KB to `~/.claude/archive-<date>/`.
> Rollback script: `bash ~/.claude/archive-<date>/rollback.sh`.
> Keep archive for at least 30 days before deleting.

## Archive retention recommendation

Default: **keep 30 days**, then user manually deletes.

Don't schedule an auto-deletion — the archive is a cheap insurance policy, and the user will have more confidence pruning it themselves when they're ready.

## Edge cases

- **A source file disappeared between Phase 0 scan and Phase 5 execution**: AutoMemory may have deleted or rewritten it. If the SHA256 doesn't match, **abort the move**, flag in chat, let the user decide.
- **Archive path collision**: Shouldn't happen with dated archive dir, but if it does, suffix with `-1`, `-2`.
- **Disk space**: AutoMemory archives are small (usually < 1 MB total), but check `df ~` before moving. Abort if disk > 95% full.
